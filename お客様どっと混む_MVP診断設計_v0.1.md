# お客様どっと混む MVP診断設計 v0.1

作成日: 2026-05-11  
対象: 那須・栃木の小規模店舗、特に飲食店・カフェを中心とした初期10店舗  
目的: Webアプリ実装前に、診断モデル・スコアリング項目・無料診断レポート・β版運用設計を固める

---

## 1. サービス定義

### 1.1 サービス名

**お客様どっと混む / okyakusa-ma.com**

意味:

- お客様.com
- お客様どっと混む
- 本来来てほしいお客様でお店が混む

### 1.2 コア価値

小さなお店が、検索・地図・SNS・口コミ・AI検索のどこから見ても、**来てほしいお客様に安心して選ばれる状態**を作る。

このサービスが売るべきものは「分析」ではなく、**選ばれる状態への翻訳**である。

### 1.3 MVPの中心体験

MVPでは「改善支援SaaS」ではなく、まず以下の体験を作る。

**えらばれる店スコア診断**

店舗オーナーが最初の1ページを見た瞬間に、

- 自分の店がネット上でどう見えているか
- どこで来店前不安が発生しているか
- SNSやGoogle Mapsを何から直せばよいか
- 自分では当たり前だと思っていた魅力が、外から見ると伝わっていないこと

に気づける診断レポート。

### 1.4 MVPでやらないこと

初期10店舗では、以下は作らない。

- ログイン付きSaaS管理画面
- SNS自動投稿
- Google Business Profile自動更新
- 食べログ・Retty等のスクレイピング
- AIだけの完全自動採点
- サブスク決済
- 競合横断ダッシュボード

ただし、**点数化・チャート化・弱点トップ3表示**はMVPでも必須とする。

| 追記済み活動報告 44日間 | 初期運用で見えた反応、投稿量、伸びる/伸びないの差 | 無料診断で再生数偏重を避け、保存・会話・地図遷移を見る |

### 2.1 世界観マーケティング

資料では、世界観は単なる雰囲気ではなく、**意味・感情・空気の一貫性**として定義されている。

診断への転用:

- 店舗の「なぜ存在するのか」が伝わっているか
- SNS、公式サイト、Google Maps、口コミの温度感が一致しているか
- 店内写真や動画に、実際の空気が写っているか
- 来店前から来店後の記憶まで、顧客の意識動線が途切れていないか

### 2.2 世界観設計7層モデル

資料の7層モデルは、店舗診断にそのまま使える。

| 層 | 診断上の意味 | 店舗診断で見るもの |
|---|---|---|
| 1. 原初物語 | なぜその店が存在するのか | 店主の思想、地域との関係、店の核となる価値 |
| 2. 世界観接触 | 最初にどう見つかるか | Google検索、Maps、SNS、外部媒体での第一印象 |
| 3. 境界侵入 | 日常から店の世界へ入れるか | 外観、入口、駐車場、予約、入店しやすさ |
| 4. 没入環境 | 店内で空気が一貫しているか | 席、光、音、接客距離、写真・動画との一致 |
| 5. 期待増幅 | 行く前の期待が高まるか | メニュー、季節性、限定性、導線動画、FAQ |
| 6. 感動体験 | 記憶に残る核があるか | 名物、所作、店主、人柄、場の時間 |
| 7. 伝播帰属 | 人に話したくなるか | 口コミ、保存、再訪理由、投稿されやすさ |

### 2.3 Save → Plan → Impulse

HCII論文およびSNS動画講座では、短尺動画の効果を以下の3段階で整理している。

| 段階 | 意味 | 主な媒体傾向 | 店舗診断で見るもの |
|---|---|---|---|
| Save | あとで行きたい、保存したい | Instagram | 世界観、空気、名物、季節感、記憶性 |
| Plan | 実際に行く前に確認したい | YouTube / Google Maps / 公式サイト | 駐車場、営業時間、価格、席、予約、子連れ可否 |
| Impulse | 今すぐ行きたい | TikTok / Shorts / 当日投稿 | 湯気、焼きたて、空席、限定、ライブ感 |

重要な示唆:

- 再生数だけでは評価しない
- Instagramの保存は「小さな来店予約」
- YouTubeは地方・車社会でPlan情報の役割が強い
- TikTok/Shortsは衝動導線だが、煽りすぎると世界観を壊す

### 2.4 感性波動論

感性波動は、商品スペックだけでなく、**お客様の感性に届く空気・所作・距離感**を重視する考え方として資料に出てくる。

診断への転用:

- 店主やスタッフの人間性が見えるか
- 店の所作、仕込み、日常が発信されているか
- 写真・動画が過剰に広告化され、店本来の空気を壊していないか
- 常連や未来のファンに届く「静かな信頼」があるか

### 2.5 二流立地構造

那須のような車移動・分散型地域では、駅前や一等地でなくても、**その場所にしかない空気**があれば目的地になる。

診断への転用:

- 不利な立地を「目的地化」できているか
- 車移動前提のPlan情報が整っているか
- 道中・周辺環境・地域性を価値に変えているか
- 「遠いけれど行きたい」理由がWeb上で見えるか

### 2.6 借景

資料では、地域の自然・歴史・観光ブランド・文化を、自店の世界観に取り込むことを借景として扱っている。

診断への転用:

- 地域・景色・観光地ブランドに乗れているか
- ただし、借景で来た客を自店の記憶に転換できているか
- 「那須に行ったついで」から「この店に行きたい」へ変換できているか

### 2.7 地域SNS

なすぱらTVの実践では、地域SNSは一店舗の宣伝ではなく、**地域全体の記憶と回遊動機を作るもの**として機能している。

診断への転用:

- 店舗単体の魅力と地域文脈が接続されているか
- 周辺スポット、道中、季節、地元の日常が見えるか
- 地域名での発見から店舗指名へ移行できるか

### 2.8 AI動画テンプレート

SNS動画講座では、最初の7パターン、拡張16パターンが整理されている。

無料診断で使う初期7パターン:

1. 商品・メニュー紹介型
2. 居心地・雰囲気型
3. 店主・スタッフの人物型
4. 仕込み・舞台裏型
5. 地域・風景・文化型
6. 共感・挑戦型
7. テロップ・無言型

診断への転用:

- 現在の投稿がどの感情を動かしているか
- Save/Plan/Impulseのどこに偏っているか
- 今すぐ撮るべき動画3本を提案できる

### 2.9 SEO/MEO/GEO

SEO/GEO資料では、AI時代には「明確・具体・一貫・根拠・文脈」が重要になると整理されている。

診断への転用:

- 店舗名、住所、営業時間、カテゴリ、メニュー、料金が明確か
- 公式サイト、Google Maps、SNSで情報が矛盾していないか
- 口コミと公式表現が一致しているか
- AIに要約されても誤解されないか
- FAQ、料金、流れ、対象顧客、向かない人が書かれているか

### 2.10 なすぱらTV実地検証

資料群では、60日間・468投稿・約300万インプレッション・約40万再生の実地検証が理論の土台として扱われている。

診断への転用:

- 点数は絶対的評価ではなく、那須で観測された行動モデルに基づく改善目安と明記する
- 地方・観光地・車移動地域では、Plan情報の不足が来店機会を大きく失わせる
- SNS動画は宣伝ではなく、未来のお客様とAIに渡す一次情報である

## 3. 診断モデル全体像

### 3.1 診断名

**えらばれる店スコア診断**

### 3.2 診断前の店舗特定判定

診断の前に、必ず「そのお店で間違いないか」を判定する。ここが曖昧なまま採点すると、同名店舗、支店、移転前情報、閉業店舗、近隣競合を誤って診断する危険がある。

#### 判定ステータス

| ステータス | 意味 | 次の動き |
|---|---|---|
| 一致 | Google Maps、公式サイト、住所、電話番号、SNSが同一店舗として整合 | 自動で診断に進む |
| 要確認 | 同名店舗、支店、住所表記ゆれ、URL不一致などがある | 候補を表示し、店舗オーナーに選んでもらう |
| 不一致 | 入力情報同士が明確に別店舗を指している | 診断を止め、入力修正を依頼 |

#### 照合項目

| 項目 | 見るもの | 判定への使い方 |
|---|---|---|
| 店舗名 | 正式名、表記ゆれ、支店名 | 同名店舗誤判定を防ぐ |
| 住所 | Google Maps、公式サイト、SNSプロフィール | 同一所在地か確認 |
| 電話番号 | Maps、公式サイト、予約サイト | 最も強い一致シグナルの一つ |
| 公式サイトURL | Maps掲載URL、入力URL | 自社サイトの同一性確認 |
| Instagram URL | プロフィールの店舗名、住所、リンク | SNS診断対象の確認 |
| 営業状態 | 営業中、閉業、移転、臨時休業 | 古い店舗情報の除外 |
| カテゴリ | カフェ、飲食、美容室、整体等 | 異業種誤判定を防ぐ |

無料診断の画面上では、細かい照合ロジックを見せすぎず、以下のように表示する。

> 店舗特定: 一致  
> Google Maps、公式サイト、住所、電話番号、Instagramが同一店舗として確認できました。

要確認の場合は、診断スコアを出す前に候補店舗を2-3件表示し、ユーザーに選択してもらう。

## 4. スコアリング項目一覧

### 4.1 スコア算出方針

点数は「客観的なランキング」ではなく、以下に基づく改善目安とする。

- 添付資料の理論
- なすぱらTV実地検証
- Google Maps/SNS/公式サイトの公開情報
- 店舗オーナー入力
- 人間レビュアーの確認
- AIによる要約・補助評価

初期MVPでは、**AI採点 60% + 人間確認 40%**を推奨する。AIだけで出すと当たっている感が薄くなり、誤採点リスクも上がる。

### 4.2 観測値と独自スコアを分ける

店主に「無理やり数値化している」と見えないよう、スコアは2種類に分けて表示する。

#### 観測値

できるだけ客観的に確認できる数値。

- 地域名×業種×目的語の検索順位
- 店舗名検索での公式サイト/Google Maps表示
- Google Mapsの口コミ件数
- Google Maps評価平均
- 口コミ最新投稿日
- 口コミ返信率
- Google Maps写真枚数
- カテゴリ、営業時間、電話、URL、メニュー、予約リンクの充足
- 公式サイトのtitle、H1、アクセス、料金、FAQ、構造化データ
- スマホ表示、表示速度の簡易チェック

公式サイト、Instagram、YouTube、TikTok、Xなど、判定対象のページが存在しない場合は、その媒体のSEO/GEO診断は **未判定** とする。存在しないこと自体を機械的に減点しない。

ただし、Google Mapsはローカル店舗診断の中核なので、Google Mapsが見つからない、または店舗特定できない場合は診断前の重要ブロッカーとして扱う。

#### 独自スコア

公開情報と店舗入力をもとに、独自診断モデルで改善目安として出す数値。

- Save導線
- Plan導線
- Impulse導線
- 来店前不安
- Instagram世界観
- 世界観一致
- 借景依存度
- 自店魅力転換度
- 顧客体験

レポート上では、観測値に「実測/確認」、独自スコアに「改善目安」とラベルを付ける。

### 4.3 SEO簡易判定辞書

SEO/MEO/GEOの簡易判定には、別紙 **SEO簡易判定辞書 v0.1** を使う。

辞書で定義するもの:

- 店舗特定の一致/要確認/不一致/未判定ルール
- 地域語、業種語、目的語
- 公式サイトのtitle、H1、FAQ、料金、アクセス等の確認項目
- Google Mapsのカテゴリ、営業時間、写真、口コミ、返信率等の確認項目
- Instagram、YouTube、TikTok、Xの簡易SEO/GEO確認項目
- 検索順位チェック用キーワードテンプレート
- 媒体がない場合の未判定ルール

無料診断では、検索順位チェックは3-5キーワードに絞る。初期βの飲食店では、なすぱらTVの実地感に基づき、まず **地域名 + グルメ / ランチ / ディナー** を優先確認する。そのうえで、業態に応じて「カフェ」「蕎麦」「ラーメン」「子連れ」「駐車場」などを追加する。

### 4.4 点数帯の意味

| 点数 | 表示ラベル | 店主への説明 |
|---:|---|---|
| 0-39 | 見えていない | 魅力がないのではなく、Web上で判断材料が足りない状態 |
| 40-59 | 伝わりかけ | 店の良さはあるが、初めての人が選ぶには不安が残る状態 |
| 60-79 | 選ばれる土台あり | かなり整っているが、弱点を直すと来店導線が強くなる状態 |
| 80-100 | 指名されやすい | 来てほしい人に見つかり、安心して選ばれる状態に近い |

## 5. 各スコアの評価基準

### 5.1 総合えらばれ度

| 項目 | 内容 |
|---|---|
| 何を見るか | 検索、地図、SNS、口コミ、世界観、来店前不安、体験記憶の総合状態 |
| 入力データ | 全入力、Google Maps、公式サイト、Instagram、口コミ傾向、人間確認 |
| 0点目安 | 店舗情報が散らばり、営業時間・価格・雰囲気・導線がほぼ判断できない |
| 50点目安 | 基本情報はあるが、来店前不安や世界観のズレが残る |
| 100点目安 | 見つかる、安心する、保存する、行く、話す、再訪する導線が一貫 |
| 店主説明 | 「お店の魅力そのものではなく、初めての人から見た“選びやすさ”の点数です」 |
| 表示 | 無料・有料 |

### 5.2 Google Maps整備度

| 項目 | 内容 |

## 6. 無料診断と有料診断の切り分け

### 6.1 無料診断で出すもの

無料診断は、**気づきと驚き**に集中する。

出すもの:

- 総合えらばれ度
- 8軸レーダーチャート
- 強みトップ3
- 弱点トップ3
- 逃している可能性のあるお客様
- 今すぐ直すべき1点
- 今撮るべき動画3本
- 主要スコアの簡易コメント
- 有料診断への導線

無料で出しすぎないもの:

- 競合比較の詳細
- 口コミの深い分類
- 世界観7層の詳細採点
- 借景/自店魅力転換の詳細分析
- 投稿カレンダー
- 具体的な改善文面、プロフィール改善案、FAQ全文

### 6.2 有料診断で出すもの

有料診断は、**改善の設計図**にする。

出すもの:

- 16項目の詳細採点
- 7層モデル評価
- Save/Plan/Impulse別の投稿棚卸し
- Google Maps改善チェックリスト
- 公式サイト/GEO改善項目
- 来店前不安の詳細リスト
- 口コミ傾向の要約
- 借景依存度と自店魅力転換度
- 30日改善アクション
- 動画企画10本
- 必要に応じてZoom解説

### 6.3 厳しめの設計判断

無料診断を完全自動にしすぎると、店舗オーナーが「よくあるAI診断」と感じる可能性が高い。初期10店舗は必ず人間確認を入れ、レポート内に**その店固有の一文**を入れる。

例:

> 「Instagramでは料理写真は伝わっていますが、初めての人が知りたい“駐車場から入口までの安心感”がまだ弱く、Plan導線で落としている可能性があります。」

この一文があるかどうかで、有料相談への関心が大きく変わる。

## 9. β版10店舗での検証設計

### 9.1 β版の目的

β版10店舗では、スコアの正しさより先に、以下を検証する。

1. 店主が「当たっている」と感じるか
2. 点数が行動につながるか
3. 無料診断から有料相談へ進むか
4. どの項目が最も刺さるか
5. 手作業をどこまでテンプレート化できるか

### 9.2 対象店舗

初期10店舗は、可能なら以下のように分ける。

| 種別 | 店舗数 | 理由 |
|---|---:|---|
| カフェ | 3 | 世界観・Saveが出やすい |
| ランチ/飲食店 | 3 | MEO/口コミ/Planが出やすい |
| 観光地寄り店舗 | 2 | 借景依存と自店転換を検証 |
| 地元密着店 | 1 | 観光ではない地域SNSを検証 |
| 美容/整体など非飲食 | 1 | 横展開可能性の確認 |

### 9.3 検証指標

| 指標 | 測り方 | 成功ライン |
|---|---|---|
| レポート初見反応 | その場で表情・発言を記録 | 10店舗中7店舗が「なるほど」「確かに」と発言 |
| 点数化への納得度 | 5段階アンケート | 平均4.0以上 |
| 当たっている項目 | 自由回答 | 3項目以上の具体発言 |
| 不快だった項目 | 自由回答 | 借景/弱点表現で強い反発がない |
| 有料診断関心 | その場で確認 | 10店舗中3店舗以上 |
| 月額支援関心 | その場で確認 | 10店舗中2店舗以上 |
| 店舗訪問診断価値 | 価格感ヒアリング | 49,800円に1店舗以上が前向き |
| 行動実施率 | 2週間後確認 | 10店舗中5店舗が1つ以上実行 |
| 改善後変化 | 30日後確認 | 投稿/Maps/口コミ/問い合わせに変化 |

### 9.4 ヒアリング質問

レポート提示直後:

1. 最初に目に入った点数・項目はどれですか
2. 「当たっている」と感じたところはどこですか
3. 逆に違和感があったところはどこですか
4. 自分では気づいていなかった弱点はありましたか

### 9.5 β版で必ず検証する仮説

仮説1: 店舗オーナーは「SEO」より「来店前不安」の方に強く反応する。  
仮説2: 「Instagram世界観」より「Plan情報不足」の指摘の方が行動につながりやすい。  
仮説3: 「借景依存度」は言い方を間違えると反発されるが、「地域の力を自店の記憶に変える」と説明すると受け入れられる。  
仮説4: 無料診断で有料化につながるのは、総合点ではなく「今すぐ直すべき1点」と「今撮るべき動画3本」。  
仮説5: 初期有料商品は月額支援より、9,800円-19,800円の詳細診断またはZoom解説付きが売りやすい。

## 11. 実装可能性を高めるための厳しめメモ

### 11.1 スコアは多すぎる

16項目は診断側の内部構造としては良いが、無料レポートの表面に全部出すと店主が疲れる。1ページ目は8軸まで。詳細ページで16項目を見せる。

### 11.2 「借景依存度」は表現注意

借景依存という言葉は、店主によっては「自分の店の力ではない」と受け取られる。無料診断では、

**地域の力を活かせている度 / 自店指名への転換余地**

のように柔らかく表示する方がよい。

### 11.3 価格は最初から月額に寄せすぎない

β版では、月額支援より先に「詳細診断」「Zoom解説付き診断」の方が売りやすい。月額は、診断後に行動できなかった店舗への受け皿として出す。

### 11.4 無料診断は自動生成感を消す

無料診断でも、最低1つはその店固有の観察を書く。ここを省くと、無料SEO診断ツールとの差別化が消える。

### 11.5 最初の勝ち筋は「Plan不足の可視化」

Instagramを頑張っている店ほど、SaveはあるがPlanが弱い可能性が高い。ここを点数化すると「おぉ」となりやすい。

### 11.6 「味・技術」は評価しない

本サービスは、店舗の本業そのものを断定評価しない。評価するのは、Web上での見え方、選びやすさ、不安の消え方、世界観の伝わり方である。この線引きをフォームとレポートに必ず書く。

## 12. v0.1の結論

お客様どっと混むのMVPは、SaaSではなく、**診断レポート型のリード獲得プロダクト**として始めるべきである。

最初の価値体験は、

> 自分のお店は、ネット上でこう見えていたのか

という驚き。

そのために、無料診断では「点数化」「レーダーチャート」「弱点トップ3」「今すぐ直すべき1点」「今撮るべき動画3本」を必ず出す。

独自性は、SEO/MEOツールではなく、以下を一つの診断モデルに統合する点にある。

- SEO/MEO/GEO
- Google Maps
- 口コミ
- Instagram/SNS動画
- Save → Plan → Impulse
- 来店前不安
- 世界観7層モデル
- 借景と自店魅力転換
- 顧客体験
- なすぱらTV実地検証

初期10店舗では、このモデルが「当たっている」と感じられるか、そして実際に行動が変わるかを検証する。

最初に作るべきものはWebアプリではなく、**診断スプレッドシート、レポートテンプレート、入力フォーム、AI下書きプロンプト、βヒアリングシート**である。
